# EntityFrameworkDemo
A project to explore all the entity framework features

# Problem statement used 
- Design a Schema(Entities) to store Organization details.
An Organization can have Employees and Customers
- Follow Single Responsibility Principle
- Have a Domain driven design helping to avoid object-relational impedance mismatch problem
- Add a support for transactions as well
- Design classes to be independently testable (Unit testing)

# Libraries used
- .Net core 8
- EntityFrameworkCore
- dotnet ef migrations
- Automapper
- xUnit
- Moq
